# 干部信息管理系统

一套面向组织人事场景的干部信息管理系统：由干部组集中维护干部信息，多科室在局域网内同步查看、搜索、导出，并支持到龄提醒、批量维护、数据分析等。

## Language

**干部**：
系统管理的核心对象，即一名被记录信息的人员。一条干部记录聚合其全部档案字段（基本信息、学历、职务、奖惩、家庭成员等）。
_Avoid_：员工、职员、人员、用户（"用户"专指系统登录账号，不是被管理的对象）

**科室**：
使用本系统的组织单元。每个科室对应一个或多个登录账号，账号带角色。
_Avoid_：部门（统一用"科室"）

**干部组**：
负责集中维护干部信息的那个科室，是"维护者"角色的典型持有者。

**维护者**：
对干部信息有增、删、改权限的角色，由干部组持有。
_Avoid_：编辑、管理者（"管理者"易和系统管理员混淆）

**查看者**：
只能搜索、查看、导出干部信息，不能修改的角色，由其他相关使用科室持有。
_Avoid_：只读用户、访客

**管理员**：
管理登录账号与角色分配的角色，管的是"账号"而非"干部档案数据"。与"维护者""查看者"是正交的两件事。

**操作留痕 / 操作日志**：
记录"谁、何时、对哪条干部记录、改了什么"的审计轨迹。因干部数据敏感，任何修改都应留痕。

**到龄**：
干部达到其法定退休年龄。退休年龄规则随性别/身份/政策而变，是一条独立待厘清的规则，不是固定数字。

**精准搜索**：
按字符精确比对的匹配方式。"张锋"与"张峰"是不同的字，必须返回不同结果——**不做**拼音、同音字、近义的模糊归并。是本系统搜索的默认且唯一语义。
_Avoid_：模糊搜索、智能搜索、拼音搜索

**简历**：
干部的一段学习/工作经历长文本。区别于其他扁平字段，它支持"用词条搜索某一段简历"，即对该文本做关键词全文检索。

**家庭主要成员及社会关系**：
挂在一条干部记录下的可重复子表，每行含 称谓、姓名、出生年月、政治面貌、工作单位及职务。一名干部对应多行。

## Flagged ambiguities

- **"同步"**：需求原文"信息变更及时同步"指"其他科室能及时看到干部组所做的变更"，即同一数据源的实时可见，**不是**多份数据库之间的复制同步。
