# 存储改用纯 Go 的 JSON 文档存储（取代 ADR-0004 中的 SQLite）

确认服务器需支持 Windows 7 后，编译须用 Go 1.20（最后支持 Win7 的版本）。而纯 Go 的 SQLite 驱动（modernc.org/sqlite）较新版本要求 Go ≥ 1.21，与 Win7 约束冲突；带 CGO 的驱动又会引入 C 运行时依赖，违背"单文件、零依赖、离线"目标。考虑到一个单位的干部数据量很小（百到数千条、写入并发极低），改用**纯 Go 的内存 + JSON 文件存储**：全部数据载入内存，每次修改以"临时文件 + 原子改名"落盘，读写以读写锁保护。

本决策取代 ADR-0004 中"数据库 = SQLite"一条，其余（Go 单文件、Inno 一键安装、离线分发）不变。

## Consequences

- 零外部依赖、确定能在 Win7 上跑；筛选/全文检索在内存中用 Go 实现，量级足够快。
- 备份/恢复 = 复制 / 还原一个 `zhiren.json` 文件（与 ADR-0004 的备份理念一致）。
- 数据文件默认放在所有用户可写的 `ProgramData\zhiren\data\` 下，避免装在 Program Files 时无写权限。
- 若未来数据量或并发显著增长，可再评估迁移到嵌入式 SQL 存储（届时 Win7 多半已退役，Go 版本约束解除）。
