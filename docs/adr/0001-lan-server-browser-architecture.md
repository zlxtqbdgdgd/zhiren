# 采用局域网服务端 + 浏览器客户端（B/S）架构

需求文档第 2 条要求"多科室局域网连接、信息变更及时同步"（干部组维护，其他科室同步看到），这从根本上排除了纯单机程序——单机数据各自存放，做不到跨科室实时同步。我们决定由办公局域网内一台机器运行服务端 + 数据库，其余 Windows / 华为平板设备通过浏览器访问，从而一次性解决跨平台（Win7 + 鸿蒙）、实时同步与集中备份三件事。

## Considered Options

- **局域网服务端 + 浏览器（选定）**：单一数据源，天然支持多科室实时同步；客户端零安装；备份只需备份服务端一处；gh 上发布的是"服务端安装包"，在那一台机器上装一次即可。
- **单机桌面 + 共享数据库**：每台设备装客户端连同一数据库。Win7 与华为平板需同一套桌面代码，跨平台代价高；多端安装与升级维护成本大；同步只能落在数据库层。
- **纯单机版**：最简单，但无法满足需求 2 的多科室同步，仅适合单人录入维护场景——与需求冲突，否决。

## Consequences

- 需要办公环境内有一台可长期开机、当"服务器"用的机器，并有人偶尔维护它。
- "发安装包给朋友"的对象是**服务端安装包**，不是每台终端各装一份。
