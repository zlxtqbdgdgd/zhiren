# 技术栈：Go 单文件 exe + SQLite + 轻量 Web，Inno Setup 一键安装，内置绿色浏览器

为满足"完全离线自包含、装在一台普通 Windows 办公机、备份要简单、一键安装、客户端零安装"这组约束，选定：

- **后端**：Go 编译成单个 `.exe`，内嵌 Web 服务并把前端静态资源一并打入该文件——零运行时依赖，拷贝即可运行。
- **数据库**：SQLite（单个 `.db` 文件）。科室数少、并发低、单位级数据量小，SQLite 足够；备份即复制该 `.db` 文件。
- **前端**：克制的现代 Web，兼容档对齐老 Chromium（Chrome 109 一档），不追前沿特性，不为 IE 买单。
- **打包**：Inno Setup 生成一个 Windows 安装包，内含 服务端 exe + 初始 db + **绿色（免安装）Chromium 浏览器** + 桌面快捷方式（用该浏览器打开 `http://本机:端口`）+ 注册为开机自启的 Windows 服务。
- **构建/发布**：GitHub Actions 构建安装包并发布到 GitHub Releases，由维护者下载后经移动介质带入内网交付（见 ADR-0002）。

## Considered Options

- .NET：Win7 上需带运行时，自包含发布体积大。
- Java：需带 JRE，离线包更重。
- Electron 桌面：属桌面方案，违背 B/S（ADR-0001）。
- 选 Go：单文件、零依赖、离线最省心。

## Consequences

- **内置绿色 Chromium 浏览器**（随安装包打入），桌面快捷方式用它打开访问地址，从根上消除"现场浏览器太旧/只有 IE"的风险，用户也无需自行安装浏览器。残留风险：物理隔离/涉密内网若有软件白名单，需确认允许运行该绿色浏览器。首页仍保留 IE 检测提示作兜底（万一有人仍用系统 IE 直接访问）。
- 若充当"服务器"的机器是 Win7，须用最后支持 Win7 的 Go 版本（Go 1.20）编译。
- 升级靠重新分发安装包并替换 exe，`.db` 数据文件保留不动。
